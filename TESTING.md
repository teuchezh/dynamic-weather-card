# Инструкция по тестированию карточки в браузере

## Быстрый способ

1. Установите простой HTTP-сервер (если еще не установлен):

```bash
# Python 3
python3 -m http.server 8000

# Или Node.js (если установлен http-server)
npx http-server -p 8000

# Или PHP
php -S localhost:8000
```

2. Откройте в браузере:
```
http://localhost:8000/test.html
```

## Важно!

⚠️ **Нельзя открывать `test.html` напрямую через `file://`** - это не будет работать из-за ES6 модулей и CORS ограничений. Нужен локальный HTTP-сервер.

## Альтернативный способ - через VS Code

1. Установите расширение "Live Server" для VS Code
2. Правой кнопкой на `test.html` → "Open with Live Server"

## Что можно тестировать

- ✅ Все погодные условия (солнце, дождь, снег, туман, гроза, облака)
- ✅ Смена времени суток (день, ночь, восход, закат)
- ✅ Различные параметры (температура, влажность, скорость ветра)
- ✅ Изменение высоты карточки
- ✅ Плавность анимаций

## Использование тестового файла

1. Выберите погодное условие из выпадающего списка
2. Установите параметры (температура, влажность, ветер)
3. Выберите время суток (или оставьте "Автоматически")
4. Нажмите "Обновить карточку"
5. Наблюдайте анимацию!

## Решение проблем

### Карточка не загружается
- Убедитесь, что запущен HTTP-сервер
- Откройте консоль браузера (F12) и проверьте ошибки
- Проверьте, что файл `animated-weather-card.js` находится в той же папке

### Анимация не работает
- Проверьте, что JavaScript включен в браузере
- Убедитесь, что браузер поддерживает Canvas API
- Попробуйте обновить страницу

### Зависимости не загружаются
- Проверьте подключение к интернету (Lit загружается с CDN)
- Если возникают ошибки с `@lit/reactive-element`, попробуйте:
  1. Обновить страницу (Ctrl+F5 для полной перезагрузки)
  2. Очистить кеш браузера
  3. Проверить консоль браузера на другие ошибки загрузки модулей
- Если проблема сохраняется, можно попробовать использовать esm.sh вместо unpkg:
  В файле `test.html` замените все `unpkg.com` на `esm.sh` в секции importmap
- Если нет интернета, можно скачать Lit локально и изменить пути в `test.html`
